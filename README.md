# docs-init

ドキュメント管理のための初期化テンプレートプロジェクト。知見とルールを体系的に蓄積・管理するための基盤を提供します。

## 概要

このプロジェクトは、開発プロジェクトにおけるドキュメント運用の標準化を目的としています。`docs/` ディレクトリを唯一のソース・オブ・トゥルース（SoT）として、知見とルールを構造化して管理します。

## ディレクトリ構造

```
docs-init/
├── AGENTS.md              # エージェント運用ルール（参照優先順位、PDCAフロー）
├── docs/
│   ├── INDEX.md          # ドキュメント索引と運用ルール
│   └── dev/
│       └── branch.md     # ブランチ戦略とCI/CDワークフロー
├── .claude/
│   └── CLAUDE.md         # Claude設定とdocs連携ルール
└── README.md             # 本ファイル
```

## 運用ルール

### 参照優先順位

1. `AGENTS.md` - 運用ルール
2. `.claude/CLAUDE.md` - docs運用・命名規約
3. `docs/INDEX.md` - 最新の知見と配置

### ドキュメント配置原則

- `docs/` は唯一のSoT。直下に置けるのは `docs/INDEX.md` のみ
- その他のドキュメントはサブディレクトリへ配置
- サブディレクトリは必要最小限にし、`kebab-case` 命名を徹底
- 追加・更新時は必ず `docs/INDEX.md` を改訂
- 機密情報（PII等）は保存禁止

### 作業フロー（PDCA）

1. **PLAN**: ルール確認とToDo化から開始
2. **DO**: 小さな単位で実装し、対応ドキュメントを即時更新
3. **CHECK**: リンク切れ、命名不整合、重複を確認
4. **ACTION**: 改善点と知見を `docs/` に蓄積し、必要ならルール更新提案

## セットアップ

このテンプレートを新しいプロジェクトに適用する場合：

1. 本リポジトリをクローンまたはダウンロード
2. プロジェクトのルートディレクトリに配置
3. `docs/INDEX.md` をプロジェクトの実情に合わせて更新
4. 必要に応じて `AGENTS.md` や `.claude/CLAUDE.md` をカスタマイズ

## コミット方針

- ドキュメント更新: `DOC: ...`
- コード変更: 目的がわかる短いメッセージ
- 不要な改行・空白調整のみのコミットは避ける
